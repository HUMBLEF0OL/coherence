## Theme
**v0.1 reacts. v0.2 proposes. v0.3 distributes.**
Coherence moves from a single-developer, single-machine plugin into something installable from a marketplace, configurable by a team, scope-aware in a monorepo, and able to surface what other developers are doing. v0.3 is the first release where "who else uses this plugin" becomes a first-class question.
---
## Problem statement
v0.1 healed existing anchored docs. v0.2 made the plugin proactive within a single developer's session. Four classes of problem remain after v0.2 ships:
1. **No distribution path.** The plugin is currently install-from-source. Without a marketplace target, plugin.json shape, signing, and update UX, v0.2 cannot reach users at scale.
2. **No team substrate.** `coherence/ignore`, mode graduations, and proposal queues are per-developer. Two teammates working in the same repo silently duplicate work, and decisions made by one are invisible to the other.
3. **No monorepo scope.** Nested [CLAUDE.md](http://CLAUDE.md) files have no inheritance contract. The plugin treats every [CLAUDE.md](http://CLAUDE.md) as global, which breaks at the first monorepo with package-level conventions.
4. **Carry-overs from v0.2.** Egress (DD-086), de-annotate (DD-069 reverse), per-file scan tombstones (DD-066 reservation), and the conditional Author-pipeline planner (DD-067 staged adoption) all park at v0.3.

## Load-bearing premises (carry-over from v0.1/v0.2)
- **DD-065 trust model is transitive.** A proposal authored on teammate A's machine and pulled by teammate B remains quarantined under `.claude/coherence/proposals/`. Net-new files never auto-land — even when they cross machines.
- **Single privacy boundary remains ****`coherence/ignore`** (NFR-PRIVACY-N2). Team-shared ignore is *additive*, never bypassing.
- **Single coordinated migrator** (DD-080 precedent): one `v2_to_v3.ts`, not per-file.
- **`prompts/v1/`**** + ****`prompts/v2/`**** ship side-by-side** (DD-091, NFR-MAINT-N2). v0.3 inherits.
- **Quarantine directory ****`.claude/coherence/proposals/<kind>/<id>/`** (DD-072) was already chosen with the team-share story in mind (see OQ-v2-06 rationale).

<span color="green">**Architectural commitments 2026-05-10 (amends Non-goals + Sequencing gates)**</span>
- **Non-goal: backend / database / hosted upload service — ever.** (DD-117) cohrence is a file-only plugin in perpetuity. Plan storage uses git (`coherence/plans/`); telemetry export uses local JSONL + user-driven curl. Server-backed alternatives are permanently rejected, not deferred.
- **Non-goal: legacy version support.** (DD-118) Each major version stands alone. v0.3 ships fresh state on install — no v1→v2 / v2→v3 migrator, no `prompts/v1/` in the tarball, no rollback to a prior major version. Major-version bumps may break the on-disk format; users re-install rather than migrate.
- **Sequencing gates collapsed.** No v0.2.0 GA tag step (no audience), no v0.2-alpha period, no v0.2.1 separate release. Gate #1 = tag and ship v0.3 directly. Gate #2 = corpus calibration passes M-CALIB-1 floor (folded into v0.3 ship-time acceptance, not a separate v0.2.1 release). Gates #3/#4/#5 retired or closed by DD-093/DD-099. See [BRD-5 Roadmap & Post-GA Commitments](https://www.notion.so/35c010d46a70815e9648e842493b141b) for the simplified gate list.
- **Marketplace status.** Listing request raised on Anthropic plugin registry. Package name on registry = `cohrence` (matches package.json + plugin.json after the rename). npm channel optional / squat-prevention only — not maintained.

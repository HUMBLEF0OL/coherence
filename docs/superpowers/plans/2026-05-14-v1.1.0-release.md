# v1.1.0 Release Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Cut, sign, and publish v1.1.0 — the single tagged release that contains all five phases of the S+ roadmap (Phases 1–5).

**Architecture:** Bump version across all sources (using `npm run bump` from Phase 1 Task 3), draft release notes, cut a `v1.1.0-rc.1` pre-release tag, run the 3-tester window (~1 week), absorb feedback into patches, cut the GA tag, fast-forward staging + master, push everything. CI install-smoke (D3) and SLSA-L3 provenance (T5) fire automatically on the tag push.

**Tech Stack:** Same as the rest. Cosign for signing. The release-engineering moves from Phase 1 (M3 — derived TAG, `npm run bump`) and Phase 4 (T8 — auto-generated release notes, T5 — SLSA, T6 — SBOM) all converge here.

**Spec:** [docs/superpowers/specs/2026-05-14-s-plus-roadmap-design.md](../specs/2026-05-14-s-plus-roadmap-design.md)

**Depends on:** All five phase plans landed on `dev`. This plan is the *last* thing that runs.

## Self-review

Mapping to spec § Release:
- Spec said "Phase 1 ships as v1.1.0, Phase 2-5 follow in v1.1.x/v1.2.0+". The user overrode that — all phases ship as one v1.1.0. This plan reflects that decision.
- The 3-tester window from Phase 2's S6 was originally a 2-week inter-phase loop; now it's a 1-week pre-release validation window (Task 2).
- The rc1 → GA flow lets us absorb tester feedback without a forever-frozen tag.

Failure modes considered:
- A workflow fails on the rc tag → fix-forward, re-tag rc2.
- A workflow fails on the GA tag → if not announced, fix-forward and force-move the tag (rare); if announced, cut v1.1.1 instead.
- Fast-forward fails because someone landed a commit on master directly → stop, investigate (master has been hands-off for the v1.0.3+ phases; an unexpected commit means human-on-call).

No placeholders. Each release-gate step has an explicit Run command and Expected output.

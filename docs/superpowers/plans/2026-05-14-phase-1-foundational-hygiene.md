### Step 3.2 тАФ Implement the bump script

- [ ] **Step 3: Create `scripts/bump.mjs`**

Create `scripts/bump.mjs` with:

```javascript
#!/usr/bin/env node
/**
 * Atomic version bump across all 7 sources of truth.
 *
 * Usage:
 *   node scripts/bump.mjs <new-version>
 *
 * Sources updated (must all stay in lockstep тАФ enforced by
 * scripts/release-ga.mjs#assertVersionSync):
 *   1. package.json#version
 *   2. package-lock.json#version
 *   3. .claude-plugin/plugin.json#version
 *   4. .claude-plugin/marketplace.json#plugins[0].version
 *   5. .claude-plugin/marketplace.json#plugins[0].source.ref  (prefixed with "v")
 *   6. src/state/init.ts#PLUGIN_VERSION literal
 *   7. README cosign-verify block тАФ regenerated by `npm run build`
 *      via scripts/render-readme-verification.mjs
 */
import { readFileSync, writeFileSync } from 'node:fs';
import { fileURLToPath } from 'node:url';
import path from 'node:path';

const SEMVER_RE = /^\d+\.\d+\.\d+(?:[-+][\w.-]+)?$/;

const DEFAULT_PATHS = {
  packageJsonPath: 'package.json',
  packageLockPath: 'package-lock.json',
  pluginJsonPath: '.claude-plugin/plugin.json',
  marketplaceJsonPath: '.claude-plugin/marketplace.json',
  initTsPath: 'src/state/init.ts',
};

export function bumpAllSources(version, paths = DEFAULT_PATHS) {
  if (!SEMVER_RE.test(version)) {
    throw new Error(`Not a valid semver string: ${version}`);
  }

  const pkg = JSON.parse(readFileSync(paths.packageJsonPath, 'utf8'));
  pkg.version = version;
  writeFileSync(paths.packageJsonPath, JSON.stringify(pkg, null, 2) + '\n');

  const lock = JSON.parse(readFileSync(paths.packageLockPath, 'utf8'));
  lock.version = version;
  if (lock.packages?.['']) lock.packages[''].version = version;
  writeFileSync(paths.packageLockPath, JSON.stringify(lock, null, 2) + '\n');

  const plugin = JSON.parse(readFileSync(paths.pluginJsonPath, 'utf8'));
  plugin.version = version;
  writeFileSync(paths.pluginJsonPath, JSON.stringify(plugin, null, 2) + '\n');

  const market = JSON.parse(readFileSync(paths.marketplaceJsonPath, 'utf8'));
  market.plugins[0].version = version;
  market.plugins[0].source.ref = `v${version}`;
  writeFileSync(paths.marketplaceJsonPath, JSON.stringify(market, null, 2) + '\n');

  const initTs = readFileSync(paths.initTsPath, 'utf8');
  const updated = initTs.replace(
    /(export const PLUGIN_VERSION\s*=\s*)['"][^'"]+['"]/,
    `$1'${version}'`,
  );
  if (updated === initTs) {
    throw new Error(`Failed to find PLUGIN_VERSION literal in ${paths.initTsPath}`);
  }
  writeFileSync(paths.initTsPath, updated);
}

if (fileURLToPath(import.meta.url) === path.resolve(process.argv[1] ?? '')) {
  const target = process.argv[2];
  if (!target) {
    console.error('Usage: node scripts/bump.mjs <new-version>');
    process.exit(1);
  }
  bumpAllSources(target);
  console.log(`[bump] all 7 sources now at ${target}. Run \`npm run build\` to regenerate README cosign block.`);
}
```

### Step 4.4 тАФ Write the migration guide

# v1.1.0 migration: `cohrence` тЖТ `coherence`

The npm slug and plugin name changed from `cohrence` to `coherence` in v1.1.0. This is an alias change тАФ no behavior changes, no schema changes. But the install slug is different, so existing marketplace installers need to re-add the marketplace under the new name.

## Marketplace users

```bash
# remove the old slug (uninstalls plugins from it)
claude plugin marketplace remove cohrence

# add under the new name
claude plugin marketplace add HUMBLEF0OL/coherence@v1.1.0

# re-install the plugin
claude plugin install coherence@coherence
```

Your project's `coherence/` state directory (proposals, trust ledger, metrics) is untouched. Plugin data at `~/.claude/plugins/data/...` is keyed off the new plugin slug; if you had local-only plugin state under the old slug, it isn't auto-migrated. Most users won't notice.

## Why minor and not major?

Semver-wise this is an alias change, not a behavior break. The two-stage pipeline, trust ladder, asserts engines, hook surface, and asserts schema are all unchanged. We're calling it a minor because functionality is preserved end-to-end.

## What's NOT changing

- Slash command prefix is still `/coherence:` (it always was тАФ the brand never spelled it `cohrence`)
- Anchor format, asserts schema, hook events, file layout under `coherence/` тАФ all stable
- v1.0.x users can leave their projects' committed state as-is
```

# Coherence v1.0 — CHANGELOG

> v1.0 is the **trust + intelligence** release: per-section trust ladder,
> cross-session learning, `asserts:` frontmatter, deeper audit + metrics, and
> signed release artifacts via Sigstore cosign keyless OIDC.

## M0 — Trust ledger foundation (TS-2, DD-138)
- `src/state/trustLedger.ts` — per-developer, per-section accept/edit/revert event log
  at `.claude/coherence/trust-ledger.json` (gitignored, survives plugin re-install).
- DD-138 weighted score formula with 30-day half-life (`ALPHA = 0.977`),
  numerator/denominator weights `accept=+1/+1`, `revert=-1/+1`, `edit=0/+0.5`.
- LRU cap at 200 events per section, sorted ascending by `_ts`.
- `recordEvent()`, `getSectionScore()` (lazy recompute on stale summary),
  `checkPromoteEligibility()`, `listOrphanedKeys()`, `reassociateKey()`,
  `expireOrphans()`.
- AJV-validated `trust-ledger.schema.json` registered with StateStore (schema_version 3).
- Edge cases hardened: empty-ledger init (FR-LEDGER-5), schema-version mismatch
  fail-closed (Edge #12), clock-skew clamp (Edge #11).

## M1 — Trust ladder (TS-3, TS-5)
- `/coherence:trust` — 5-section status report, `sync`, `--promote --auto-land <kinds>`,
  `--prune-stale --yes`.
- `src/state/teamAggregate.ts` — `coherence/trust/<author-hash>.json` per-developer
  committed files, 180-day staleness filter, contested flag derivation
  (FR-LEDGER-4 pass-3 threshold `|aggregate| < 0.2` with ≥ 2 contributors).
- Trust-score gate in stop pipeline: destructive + frontmatter ALWAYS defer to
  confirmation; modifying requires score ≥ 0.85 for sectionRef; additive
  auto-applies (v0.2 carry).
- SessionStart one-time promote hint (FR-TRUST-1), stamped via
  `promote_hint_emitted_at`.
- `proposal_accept_recorded`, `proposal_edit_recorded`, `proposal_revert_recorded`,
  `trust_promoted`, `trust_synced` telemetry event types (FR-TELEMETRY-1).
- `isKindAutoLandable()` helper for the v0.2 quarantine boundary relaxation.

## M2 — `asserts:` validation pipeline (TS-4, FR-ASSERTS-*)
- `src/validation/assertions/` — registry + 5 text-pattern engines
  (`has_example`, `no_placeholder_links`, `max_words`, `min_words`,
  `no_todo_comments`) + 2 codebase-linked engines (`symbol_exists`,
  `file_exists`).
- Codebase-linked engines use `fast-glob` + batched parallel reads with
  short-circuit (pass-1 amendment); per-session file-list cache by language.
- Per-assertion `policy: 'block' | 'warn'` (default `warn`).
- 10-assertion-per-section cap + one combined stderr warning per session
  (FR-ASSERTS-1 + pass-1 Minor #6).
- Unknown assertion types logged + ignored (FR-ASSERTS-5).
- Integration into `src/pipeline/stage2.ts` after the hallucination check.

## M3 — Metrics + deep audit (TS-5, TS-6)
- `/coherence:metrics` — 5-section Markdown report: summary, top drifting,
  trust scores (personal + team aggregate), Unicode-sparkline cost trend,
  revert hotspots with configurable `--revert-threshold` percent.
- `--out <path>` writes via DD-128 sandbox helper; `--allow-out-of-tree` flag
  for opt-in escape; `--since YYYY-MM-DD` window filter.
- `/coherence:audit` extended:
  - Free tier: doctor + scope-debug + status + metrics export + token budget
    classification (`Normal` < 2000, `Large` 2000..5000, `Bloated` > 5000
    tokens; `ceil(chars/4)` estimate).
  - `--deep` tier: flag-based confirmation (`--confirm-deep <signature>` or
    `--no-confirm` in CI). 12-char sha256 signature over the candidate pair
    list; stale signature throws to force re-estimate.
- `src/audit/sectionSymbolIndex.ts` — lazy-built symbol-to-sections cache at
  `.claude/coherence/section-symbol-index.json`, invalidated on
  `(section-index.json, registries/*.ts)` hash change.
- `cost-ledger.schema.json` extended with `audit_deep` stage enum.

## M4 — Trust signals + repair extensions (TS-7)
- `SECURITY.md` — coordinated disclosure policy, GitHub Security Advisories
  fallback, supported-versions matrix.
- `## Verification` README block — regenerated by `scripts/render-readme-verification.mjs`
  from `package.json#repository.url` (keeps fork-friendly).
- `.github/workflows/release.yml` — cosign keyless OIDC sign step with
  `id-token: write` + `contents: write` permissions (pass-2 Important #1).
- `scripts/release-ga.mjs` extended with `runSignStep()`: requires CI or
  `--unsigned` (renames to `<name>-<version>-UNSIGNED.tgz` for local builds).
- `release-artifacts/<name>-<version>.sha256` written alongside the tarball.
- `/coherence:repair` extended:
  - Default flow now also lists orphaned trust-ledger keys with numbered output.
  - `--reassociate <oldRef> --to <newRef>` — atomic key move + coherence-log
    entry.
  - `--expire-orphans` — bulk-removes orphaned keys + summary log entry with
    up to-20-refs preview + `… and N more` tail.

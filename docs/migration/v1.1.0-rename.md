# v1.1.0 migration: `cohrence` → `coherence`

The npm slug and plugin name changed from `cohrence` to `coherence` in v1.1.0. This is an alias change — no behavior changes, no schema changes. But the install slug is different, so existing marketplace installers need to re-add the marketplace under the new name.

## Marketplace users

```bash
# remove the old slug (uninstalls plugins from it)
claude plugin marketplace remove cohrence

# add under the new name
claude plugin marketplace add HUMBLEF0OL/coherence@v1.1.0

# re-install the plugin
claude plugin install coherence@coherence
```

Your project's `coherence/` state directory (proposals, trust ledger, metrics) is untouched. Plugin data at `~/.claude/plugins/data/...` is keyed off the new plugin slug; if you had local-only plugin state under the old slug, it isn't auto-migrated. Most users won't notice.

## Why minor and not major?

Semver-wise this is an alias change, not a behavior break. The two-stage pipeline, trust ladder, asserts engines, hook surface, and asserts schema are all unchanged. We're calling it a minor because functionality is preserved end-to-end.

## What's NOT changing

- Slash command prefix is still `/coherence:` (it always was — the brand never spelled it `cohrence`)
- Anchor format, asserts schema, hook events, file layout under `coherence/` — all stable
- v1.0.x users can leave their projects' committed state as-is
